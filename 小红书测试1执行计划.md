**三、统一技术路线**
- 技术栈：`React + Vite + TypeScript + 单文件 CSS variables`
- 为什么选它：静态站最快、Vercel 兼容最稳、无服务端、无数据库、Codex 20-60 分钟最好落地
- 目录结构：`src/data/questions.ts`、`src/data/results.ts`、`src/lib/scoring.ts`、`src/components/QuestionCard.tsx`、`src/components/ResultCard.tsx`、`src/App.tsx`、`src/styles.css`
- 是否需要组件库：`不需要`
- 是否需要 Tailwind：`不需要`；今晚直接写少量 CSS 更短链路
- 是否需要本地 JSON 题库：`需要`；题目和结果都本地文件维护，后续复制 SKU 只换数据
- 数据结构：题目 `id/title/options/score-or-axis`；结果 `id/title/subtitle/tags/description/shareText`
- 打分逻辑：A 用 `3 轴二分映射 -> 8 型`；B 用 `累计分数区间 -> 5 档`
- GitHub 仓库协作方式：`一个项目一个仓库`，不做 monorepo；当前目录为空，默认新建两个独立 repo 最省事
- push 到 main 后的 Vercel 自动部署路径：`Codex 生成代码 -> git commit -> git push origin main -> Vercel Git 集成自动构建并发布`
- 为什么这是最短路径：`零后端 / 零登录 / 零数据库 / 零 AI API / 零管理后台 / 同一套壳复制出更多 SKU`
- 统一页面美学：标题字体气质 `MiSans / PingFang SC 粗字重`；卡片 `24px 圆角 + 轻投影 + 细描边`；按钮 `全宽胶囊 + 实心主色 + 轻按压阴影`；商品封面图 `大字标题 + 3 个结果标签 + 手机界面`；小红书首图 `问题句 + 结果卡局部 + 低价角标`
